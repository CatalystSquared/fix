Mobile or Desktop web UI

Customers get a device-optimized experience regardless of how they access the storefront. The user interface is implemented with the Bootstrap HTML/CSS responsive design framework.

Spree storefront server

The storefront server runs on Heroku exposing both a web-based interface for customers as well as a REST/HTTP API for integration with other systems.

Spree Commerce provides everything needed to launch an eCommerce site, from payment gateway integrations to multi-location inventory control. Its built on Ruby on Rails, allowing complete customization by developers.

Salesforce synchronization

Heroku Connect synchronizes storefront data from Heroku Postgres to Salesforce:

  • As Spree Products are updated
    • Product2
    • Pricebook2
    • PricebookEntry
  • As Spree Users are updated
    • Contact
  • As Spree Orders are completed
    • Order__c
    • LineItem__c

Easily configured with a point and click UI, developers can further extend the integration with Salesforce using rate-limit free, bi-directional synchronization.

Salesforce-centric workflow

Records in Salesforce may be supplemented with buttons that invoke API calls to the storefront with the "Custom Button" feature.

For example, the Cancel Order button triggers a REST/HTTP API call to Spree immediately cancelling the order.
